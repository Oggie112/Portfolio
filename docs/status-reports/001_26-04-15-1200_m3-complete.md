# Status Report 001
**Date:** 15 April 2026  
**Project:** Portfolio  
**Milestone:** M3 Complete → M4 Starting

---

## What Happened

The portfolio site has gone from a working scaffold with placeholder content to a fully polished, content-complete product. All five sections are live with real data, the visual design is cohesive, and a longstanding bug that was silently preventing the scroll-driven background colour transitions from working has been found and fixed.

---

## Shipped

**Working scroll-driven background transitions** — as you scroll through the site, the background colour smoothly shifts between five distinct dark forest tones (deep green → teal-green → olive → muted green → slate-green). A bug where the page's own background colour was painting over the animated backdrop has been resolved.

**Redesigned contact section** — the original contact form with email delivery infrastructure has been replaced with a clean, minimal section featuring a styled email button and links to GitHub and LinkedIn. This removes unnecessary complexity (a verified domain would have been required) with no meaningful loss of functionality for a portfolio.

**Project modal** — clicking any project card opens a full detail view with description, tags, links, and repository information. This replaces the originally planned separate project pages, keeping the site as a single page.

**Real skills and project content** — the skills section now reflects the actual tech stack (TypeScript, React, Next.js, SvelteKit, Supabase, and others) with proficiency levels. Project cards are populated with real work.

**Headshot and image handling** — a photo has been added to the About section with proper error handling (the container gracefully holds its shape if the image fails to load) and correct size attributes to prevent layout shift.

**Component fixes** — the Card component had a dead prop and a conflicting CSS transition that was silently overriding the hover animation. Both have been cleaned up.

## Worth Remembering

The contact form was dropped in favour of a mailto link. The decision was driven by Resend requiring a verified domain to send from — without one, the form could only deliver to a Resend test inbox. For a portfolio, a visible email address is equally effective and significantly simpler.

The `once: false` setting on entrance animations means sections re-animate every time they scroll into view (scrolling back up will re-trigger the fade). This is intentional — it keeps the animations feeling alive with the background transitions — but is worth revisiting if it ever feels distracting.

All section background colours are defined in `useSectionBackground.ts` as plain hex values, separate from the CSS variables in `app.css`. If the palette ever needs updating, both places should stay in sync.
